## 1. 战略背景

### 1.1 问题陈述

易筑 Evo 是一套面向建筑工程行业的综合性企业管理系统，涵盖营销、设计、施工、行政、人事、财务、资产等 200+ 功能模块。随着系统功能持续丰富，用户面临以下问题：

- **学习成本高：** 新员工需要数周才能熟悉系统操作，培训依赖老员工传帮带或翻阅静态操作手册
- **操作效率低：** 用户在系统中频繁切换页面查找功能入口，平均完成一个跨部门流程需要打开多个页面
- **知识分散：** 操作手册、制度发文、公告、培训资料分散在各处，用户难以快速找到答案
- **问题重复：** 客服和运维团队平均每周收到 50+ 同类操作问题，重复解答消耗大量人力

### 1.2 产品目标

| 维度 | 目标 |
|------|------|
| 用户自助解决率 | 80% 以上的操作问题用户可通过 AI 自助解决 |
| 单次操作耗时 | 减少 50%，用户不再需要翻阅手册 |
| 客服工单量 | 减少 40% 的重复性操作咨询 |
| 新员工上手时间 | 从 2-4 周缩短至 1 周内 |
| 月活跃使用率 | 60% 以上的活跃用户每月至少使用 1 次智能助手 |

## 2. 用户画像

### 2.1 目标用户

**P0 — 一线业务用户**
- 营销人员、设计人员、施工项目人员、行政人员
- 日常高频使用系统进行操作和审批
- 痛点：记不住复杂流程路径，频繁遇到操作卡点

**P1 — 新员工**
- 入职 1-3 个月内
- 痛点：对系统完全陌生，需要频繁求助

**P2 — 管理者/领导**
- 项目负责人、部门经理
- 痛点：需要快速查看业务数据，但不熟悉报表模块

**P3 — 系统管理员/客服**
- 负责系统运维和用户答疑
- 痛点：重复回答相同的操作问题

## 3. 用户故事

### 3.1 核心问答

1. 作为**一线业务用户**，我想**用自然语言提问操作问题**，以便**不需要翻阅操作手册就能完成任务**
2. 作为**一线业务用户**，我希望**AI 回答中附带可直接跳转到相应功能页面的链接**，以便**一键到达操作位置**
3. 作为**新员工**，我想**问"如何申请出差""如何发起报销"等流程问题**，以便**快速上手日常工作**
4. 作为**项目人员**，我想**问"投标保证金流程走到哪一步了"**，以便**实时追踪业务进展**
5. 作为**管理人员**，我想**问"上季度营销立项数量"**，以便**快速获取业务数据**

### 3.2 智能引导

6. 作为**业务用户**，我想**AI 分步骤引导我完成一个跨部门流程（如"用印申请"）**，以便**不会遗漏任何环节**
7. 作为**业务用户**，我想**在填表单时遇到不懂的字段可以直接问 AI**，以便**不用退出页面去查手册**
8. 作为**营销人员**，我想**AI 根据我当前所在的页面提供上下文相关的帮助**，以便**获得更精准的解答**

### 3.3 知识管理

9. 作为**系统管理员**，我想**在后台管理知识库文档**，以便**随时更新和优化 AI 的知识来源**
10. 作为**系统管理员**，我想**查看用户常见问题统计**，以便**识别知识盲区并补充文档**
11. 作为**知识库维护者**，我想**批量上传操作手册、制度文档**，以便**快速构建知识库**

### 3.4 人机协作

12. 作为**用户**，我想**对 AI 的回答进行"有用/无用"反馈**，以便**帮助系统持续优化**
13. 作为**用户**，我想**在 AI 回答不准确时能够转人工客服**，以便**问题不被卡住**
14. 作为**客服人员**，我想**看到用户与 AI 的对话历史**，以便**快速理解问题背景，减少重复询问**

### 3.5 场景扩展

15. 作为**项目经理**，我想**AI 主动提醒我合同到期、证书到期等关键节点**，以便**不错过重要事项**
16. 作为**财务人员**，我想**问"某项目的预算使用情况"并看到图表**，以便**直观了解财务状况**
17. 作为**营销人员**，我想**AI 帮我解析招标文件中的关键要求**，以便**快速提取重点内容**
18. 作为**审批人**，我想**AI 在审批页面提供"一键审批建议"**，以便**根据历史规则快速决策**

### 4.2 模块一：对话交互

#### 4.2.1 自然语言问答

| 功能 | 说明 |
|------|------|
| 自由提问 | 用户输入自然语言问题，AI 基于知识库检索 + 大模型推理生成回答 |
| 多轮对话 | 支持上下文理解，用户可连续追问（如"怎么申请出差"→"那报销呢"）|
| 思考展示 | 展示 AI 的推理过程或知识来源，增强用户信任 |
| 深度链接 | 回答中自动嵌入系统页面链接，用户点击直达操作页 |
| 富文本回复 | 支持文本 + 图片 + 表格 + 链接的混合展示 |

#### 4.2.2 入口设计

| 入口 | 说明 |
|------|------|
| 全局浮窗 | 系统右下角悬浮按钮，点击展开对话面板，覆盖在任何页面上方 |
| 内嵌模式 | 在帮助中心页面的全屏对话模式，适合复杂问题的深度交流 |
| 右键唤起 | 用户在任意页面右键选择"问问 AI"，自动带入当前页面上下文 |

#### 4.2.3 深度链接协议

AI 回答中的每个操作步骤携带如下结构的数据，前端渲染为可点击按钮：

```json
{
  "steps": [
    {
      "title": "进入营销门户",
      "url": "/marketing",
      "description": "点击顶部导航「营销门户」"
    },
    {
      "title": "新建营销立项",
      "url": "/marketing/project/create",
      "description": "点击左侧菜单「营销立项」→「新建」"
    },
    {
      "title": "填写立项信息",
      "url": "/marketing/project/form?type=create",
      "description": "填写项目名称、预算金额等字段"
    }
  ]
}
```

### 4.3 模块二：智能引导

#### 4.3.1 上下文感知

- 自动识别用户当前页面（路由 + URL 参数）
- 判断用户在当前页面可能遇到的常见问题
- 在全局浮窗中给出上下文相关的快捷提问建议

**场景示例：** 用户打开"出差申请"表单时，AI 自动显示"需要帮您填写常用字段吗？"或"差旅标准是多少？"

#### 4.3.2 步骤化流程引导

对于复杂流程（如用印申请需 5 步），AI 拆解为步骤列表：

1. 填写用印申请单 → [去填写]
2. 部门负责人审批 → [查看状态]
3. 法务审核 → [查看状态]
4. 用印归档 → [去归档]
5. 完成 ✓

每步带跳转链接和状态标记。

### 4.4 模块三：知识库管理后台

#### 4.4.1 文档管理

| 功能 | 说明 |
|------|------|
| 文档上传 | 支持 PDF、Word、Markdown、TXT 等格式 |
| 批量导入 | 支持整批导入操作手册文件夹 |
| 文档分类 | 按业务模块（营销/设计/施工/行政/人事等）分类管理 |
| 版本管理 | 文档更新后保留历史版本，可回滚 |
| 自动索引 | 上传后自动切片、向量化、写入向量数据库 |

#### 4.4.2 知识库监控

- 查看知识库中每个文档被 AI 引用的频次
- 识别"高频提问但无知识覆盖"的盲区
- 标记过时文档，提示管理员更新

### 4.5 模块四：运营分析

#### 4.5.1 数据看板

| 指标 | 说明 |
|------|------|
| 日活/月活 | 智能助手使用人数及趋势 |
| 总提问数 | 每日/周/月提问量 |
| 问题分类 | 按模块统计高频问题 TOP 10 |
| 解决率 | 用户未转人工即解决问题的比例 |
| 平均响应时间 | AI 回复速度 |
| 反馈评分 | 用户"有用/无用"比例 |

#### 4.5.2 反馈闭环

- 用户对 AI 回答的每条反馈进入 review 队列
- 高频负面反馈触发知识库补充任务
- 管理员可手动修正回答并补充到知识库

### 5.2 核心流程：RAG 问答

```
用户输入 → 问题理解（重写/分类）
        → 向量检索（从知识库召回 Top-K 相关片段）
        → 上下文拼接（Prompt Template）
        → LLM 推理生成回答
        → 后处理（提取 Deep Link、格式化）
        → 返回前端渲染
```

## 10. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| LLM 幻觉导致错误引导 | 用户信任度下降 | RAG 严格检索 + 来源展示 + 人工审核机制 |
| 知识库更新不及时 | 回答过时 | 文档版本管理 + 过期标记提醒 |
| 用户提问超出知识库范围 | 回答质量差 | 兜底策略 + 知识盲区主动识别 + 转人工 |
| 响应速度慢 | 用户流失 | 流式输出 + 缓存高频问题 + 异步索引 |
| 数据安全/隐私 | 合规风险 | 知识库内容仅授权用户可见 + 对话日志脱敏 |

## 11. 附录：对话体验示例

```
用户：怎么申请出差？

AI：我来帮您分步骤完成出差申请 👇

① 进入行政门户
   点击顶部导航「行政门户」→ [去行政门户]

② 发起出差申请
   在「行政报销」下找到「出差申请」
   → [去新建出差申请]

③ 填写申请表单
   · 出差事由：填写出差目的
   · 预计天数：填写天数
   · 目的地：填写城市
   · 预计费用：估算金额
   填写完成后点击「提交」

④ 等待审批
```
