# PRD: 废柴打卡器 — V1 上架版本

## Problem Statement

市面上的习惯追踪类应用功能越来越臃肿——专注森林、社交广场、等级系统、社区动态……对于只想「每天打个卡就走」的用户来说，这种复杂度反而成为负担。用户需要一个 **打开 → 点一下 → 关掉**、全程不超过 3 秒的极简打卡工具。

具体场景包括但不限于：
- 记录每日排便情况，配合益生菌/饮食调整追踪改善效果
- 记录奶茶摄入频次，辅助逐步减量（一天一杯 → 两天一杯）
- 记录喝水、运动、读书、冥想等日常习惯的完成情况

## Solution

「废柴打卡器」——一款**极简、去负担**的微信小程序打卡工具。

保持轻松调侃的产品调性（"废柴"不是自嘲，而是接纳），拒绝花哨功能。核心交互路径只有两步：打开日历 → 点击打卡项。用户可在任一日期上前置/追溯打卡，月底自动生成统计报表。

V1 采用本地存储（微信 `wx.getStorageSync`），上架即用，无需登录。数据不上云，无用户系统，无社交功能。云同步、多设备、社交功能等作为 V2+ 规划。

## User Stories

1. 作为普通用户，我想打开小程序直接在日历上看到本月概览，这样我能快速了解这个月的打卡情况
2. 作为普通用户，我想点击日历上的任意日期查看该日的打卡项列表，这样我可以追溯历史或提前打卡
3. 作为普通用户，我想点击打卡项右侧的复选框切换完成/未完成状态，这样操作只需要一次点击
4. 作为普通用户，我想看到每个打卡项的连续打卡天数（🔥），这样能给我持续打卡的正向激励
5. 作为普通用户，我想在打卡项列表里添加新的打卡项（如"排便""戒奶茶"），这样打卡项可以自定义
6. 作为普通用户，我想长按打卡项将其删除，这样不再需要的打卡项可以随时清理
7. 作为普通用户，我想在统计页看到本月汇总数据（打卡天数、完成率、最长连续），这样能直观评估本月表现
8. 作为普通用户，我想在统计页看到每个打卡项的完成次数排行，这样能了解哪些项坚持得好、哪些需要加强
9. 作为普通用户，我想在统计页切换查看不同月份的数据，这样能追踪长期趋势
10. 作为排便记录用户，我想通过这个工具追踪排便频率的变化，这样能评估饮食调整（益生菌/蔬菜）的效果
11. 作为戒奶茶用户，我想记录每天的奶茶饮用情况，这样能逐步从一天一杯降到两天一杯
12. 作为新用户，我想在小程序中查看隐私政策说明，这样能确认数据安全后再放心使用
13. 作为开发者，我希望存储层有清晰的接口封装，这样未来从本地存储切换到云存储时只需改动一个模块
14. 作为开发者，我希望打卡数据和打卡项数据有合理的默认值，这样用户首次打开就能直接使用

## Implementation Decisions

### 模块设计

**Storage 存储层**（`utils/storage.js`）—— 深度模块
- 封装所有本地存储操作，对外暴露统一 API 接口
- 接口包括：
  - `getCheckinItems() / setCheckinItems(items)` — 打卡项 CRUD
  - `getRecords()` — 获取全部打卡记录
  - `saveRecord(date, itemName, checked)` — 保存单条打卡记录
  - `getRecordsForDate(date)` — 获取某日记录
  - `getRecordsForMonth(year, month)` — 获取某月记录（用于统计页）
  - `addItem(name)` / `removeItem(name)` — 添加/删除打卡项
  - `calculateStreak(itemName, fromDate)` — 从某天往前计算连续打卡天数
- 设计为深度模块的原因：接口固定后，V2 切换到云存储时只需替换该模块内部实现，页面代码无需改动

**日历打卡页**（`pages/index/index`）
- 复刻现有月历网格 + 日期选中 + 打卡标记圆点逻辑
- 所有数据操作通过 Storage 层进行
- 加载打卡项时自动计算 🔥 连续天数

**统计页**（`pages/report/report`）
- 复刻现有月度汇总卡片 + 打卡项完成排行
- 支持月份切换，数据通过 Storage 层聚合计算
- 三项核心指标：打卡天数、完成率、最长连续天数

**隐私政策页**（`pages/privacy/privacy`）—— 新增
- 静态页面，展示隐私声明
- 声明内容包括：本小程序不收集个人信息、不上传数据、所有数据存储于用户本地设备
- 满足微信小程序上架合规要求

### 数据存储策略

- V1 统一使用 `wx.getStorageSync` 本地存储
- 两把锁：`checkinItems`（打卡项列表）和 `checkinRecords`（打卡记录）
- 打卡记录格式：`{ "YYYY-MM-DD": [{ name: "运动", checked: true }, ...] }`
- 不上云、不涉及用户身份标识、不需要登录

### 合规事项

- 新增隐私政策页面作为独立 tab 页或入口
- 在 `app.json` 中注册隐私页面
- 备案信息在 PRD 之外由运营流程处理

### 产品风格

- App 名称：废柴打卡器
- 调性：轻松、调侃、去负担，保留 🔥 等表情符号
- 配色：主色 #3A99F2（蓝色）+ #52C41A（绿色完成标记）+ #FF6B35（橙色激励）
- 保持现有视觉风格不变，避免过度设计

## Testing Decisions

由于微信小程序的前端特性（UI 依赖真实设备环境），V1 阶段的测试策略为：

- **页面功能测试**：在微信开发者工具和真机上进行手动冒烟测试
- **边界条件**：测试空数据状态（首次打开）、跨月切换、连续多天打卡的连续性
- **数据一致性**：验证删除打卡项后，对应日期的打卡记录被同步清理
- **存储兼容性**：验证 `wx.getStorageSync` 在不同基础库版本下的表现

V2 引入云存储后可配套云函数单元测试。

## Out of Scope

- **云数据同步/多设备**：V2 规划
- **用户登录/注册/OpenID 体系**：V2 规划
- **社交功能（排行榜/好友打卡/分享到群）**：V3 规划
- **高级数据可视化（图表、趋势线）**：V3 规划
- **推送通知/每日提醒**：V2 规划
- **打卡项分类/分组/排序**：暂不追加
- **打卡项备注/笔记功能**：暂不追加
- **暗黑模式/主题切换**：暂不追加
- **数据导出（Excel/CSV）**：暂不追加
