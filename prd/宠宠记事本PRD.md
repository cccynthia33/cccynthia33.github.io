# PRD: 宠宠记事本 v1.0

## Problem Statement

多宠家庭在日常生活中面临大量琐碎的宠物看护信息管理需求：每只宠物的喂食品牌和口味偏好、排泄是否正常、驱虫和疫苗是否按时等。仅靠大脑记忆或零散的聊天记录很容易遗忘或混淆，尤其是当家中同时饲养猫和狗时，不同物种的喂养习惯和健康管理周期差异更大。

目前市面上缺少一款足够轻量、专注于"多宠日常记录"而非"社交/电商/医疗"的工具。用户需要一个开箱即用、交互简洁的备忘录式应用，能够快速记录每只宠物的日常，并在首页一目了然地看到今日概况。

## Solution

**宠宠记事本** 是一款微信小程序，为多宠家庭提供轻量级的日常记录工具。核心设计原则：

- **单用户免登录** — 直接使用微信身份，无注册流程
- **多宠切换** — 首页可快速切换当前查看的宠物，数据按宠物隔离
- **快速记录** — 喂食和排泄记录均可在两步内完成
- **今日概况** — 首页顶部展示今日喂食次数、排泄次数、待办提醒
- **本地存储** — 所有数据存储在微信本地缓存，即开即用，无需网络

## User Stories

1. 作为宠物主人，我想在首页看到今日每只宠物的喂食/排泄统计，以便快速掌握今天的照顾情况
2. 作为宠物主人，我想在首页顶部切换不同的宠物，以便查看不同毛孩子的专属信息
3. 作为宠物主人，我想快速记录喂食的品牌、口味和喜好程度，以便追踪哪款粮毛孩子最爱吃
4. 作为宠物主人，我想记录排泄的便便和尿尿状态，以便及时发现消化健康问题
5. 作为宠物主人，我想查看所有历史记录并可按宠物筛选，以便回顾某只宠物的长期情况
6. 作为宠物主人，我想在历史记录中区分喂食和排泄记录，以便快速定位特定类型的记录
7. 作为宠物主人，我想添加新的宠物档案（名字、头像、品种、生日），以便管理所有毛孩子
8. 作为宠物主人，我想编辑和删除宠物信息，以便及时更新档案
9. 作为宠物主人，我想设置定期提醒（驱虫、疫苗、洗澡等），以便不再错过重要事项
10. 作为宠物主人，我想开关提醒的有效状态，以便临时关闭不需要的提醒
11. 作为宠物主人，我想通过滑动删除不需要的提醒，以便保持提醒列表干净
12. 作为宠物主人，我想看到首页的快捷入口（喂食、排泄、提醒），以便快速跳转到常用功能
13. 作为宠物主人，我想在添加记录时选择具体时间，以便精确记录喂养/排泄时刻
14. 作为宠物主人，我想给每条记录添加备注，以便补充特殊观察（如"全吃光"、"软便"）
15. 作为宠物主人，我想在"我的"页面看到宠物数量、总记录数、提醒数量的统计，以便了解整体使用情况
16. 作为宠物主人，我想通过底部导航栏在首页、记录、管理、我的之间切换，以便快速访问各个模块
17. 作为宠物主人，当我还没有添加宠物时，首页显示引导空状态，以便我快速上手添加第一只宠物
18. 作为宠物主人，我想在喂食记录中通过 emoji 选择器快速标记喜欢程度（喜欢/一般/不喜欢），以便直观记录偏食情况
19. 作为宠物主人，我想在首页看到即将到来的提醒事项，以便提前安排
20. 作为宠物主人，我想查看"关于"页面了解应用版本信息，以便知道当前使用的版本

## Implementation Decisions

### 架构

- **全局数据层**：所有数据集中存储在 `app.globalData` 中，通过 `app.js` 提供的 save/get 方法统一读写。页面间不直接传递数据对象，改由全局状态管理。
- **持久化**：`wx.setStorageSync` 做本地缓存持久化，在每次数据变更时同步写入。应用启动时在 `onLaunch` 中从缓存恢复数据。
- **路由方式**：底部 Tab（首页/记录/管理/我的）使用 `wx.redirectTo` 切换以避免页面栈堆积，表单页（喂食/排泄/宠物编辑/提醒）使用 `wx.navigateTo` 入栈。

### 数据模型

- 使用**单一 `records` 数组**存储所有类型的记录（喂食、排泄），通过 `type` 字段区分。这是有意为之的浅模块设计——简化查询逻辑，历史列表只需一次过滤即可展示混合时间线。
- 宠物使用 `id: number`（`Date.now()` 生成）作为主键，记录和提醒通过 `petId` 关联宠物。
- 数据模型未使用规范化关系数据库设计，而是采用适合小程序端内存操作的扁平数组结构。

### 页面模块

| 模块 | 职责 | 数据依赖 |
|------|------|----------|
| `index` | 宠物切换栏、今日统计卡片、快捷记录入口、即将到来的提醒 | pets, currentPetId, records, reminders |
| `feeding` | 喂食记录表单（选择宠物、品牌、口味、喜好、时间、备注） | pets, records |
| `excretion` | 排泄记录表单（选择宠物、便便状态、尿尿状态、时间、备注） | pets, records |
| `history` | 混合历史记录列表，支持按宠物和日期筛选 | pets, records, reminders |
| `pet` | 宠物 CRUD 表单（底部弹出 Modal 形式） | pets |
| `reminder` | 提醒列表（滑动删除、开关切换）、添加提醒表单 | pets, reminders |
| `manage` | 管理快捷入口汇总（宠物管理、提醒管理、数据备份占位等） | - |
| `profile` | 数据统计概览、关于信息 | pets, records, reminders |

### UI 设计

- 设计系统定义在 `app.wxss`：暖橙色 `#FF8C42` 为主色，搭配柔和米白背景 `#FFF8F2`
- 每只宠物配置 emoji + 主题色，在宠物选择栏中通过颜色区分
- 底部 `tab-bar` 为公共组件，通过 `current` 属性高亮当前 Tab
- 所有表单页使用自定义导航栏（`navigationStyle: "custom"`），统一返回按钮和标题布局

## Testing Decisions

由于该项目为个人使用的微信小程序 MVP 版本，且无自动化测试基础设施，当前阶段采用以下测试策略：

- **人工冒烟测试**：每页在真机或开发者工具中走通主流程（添加/查看/删除）
- **边界情况验证**：空宠物列表、切换宠物后数据隔离、清除缓存后重新启动
- **不引入单元测试框架**：v1.0 阶段不引入测试框架和 CI，后续迁移云开发后再考虑
- **测试重点**：
  - 首页数据聚合（今日统计、提醒）在切换宠物后是否正确刷新
  - 记录写入后历史列表能否即时展示
  - 删除宠物后记录是否保留（当前设计：删除宠物不删除相关记录，仅不再显示在宠物选择栏）

## Out of Scope

以下功能明确不在 v1.0 范围内，计划在 v2.0 迭代：

- **数据备份与导出** — 将记录导出为 JSON/CSV 文件保存到手机
- **健康报告** — 生成周报/月报，分析喂食偏好趋势和排泄异常频率
- **体重记录** — 记录并展示体重变化曲线
- **云开发迁移** — 将数据从本地缓存迁移到微信云开发，实现多设备同步
- **照片上传** — 为记录附加照片（使用云存储）
- **主题切换** — 自定义界面主题色
- **疫苗接种时间线** — 可视化展示疫苗和驱虫的完整时间线
- **推送通知** — 通过微信订阅消息发送提醒通知

## Further Notes

- 该小程序为纯个人使用场景设计，不需要用户系统、权限管理和多用户隔离
- 所有数据存储在微信本地沙盒中，卸载小程序或清除微信缓存会导致数据丢失——用户需注意这一点，这也是 v2.0 优先做数据备份和云迁移的原因
- 宠物类型当前支持猫和狗，数据结构预留了扩展性（`type` 字段），后续可扩展支持兔子、仓鼠等其他宠物
- 排泄记录使用了专门的视觉分类（emoji 搭配状态标签），帮助用户更直观地判断排泄是否异常
- 项目的 `agent.md` 文件和此 PRD 保持了同步，作为 AI 辅助开发时的上下文知识库
