# PRD: 宠宠记事本 v1.1

## Problem Statement

多宠家庭在日常生活中面临大量琐碎的宠物看护信息管理需求：每只宠物的喂食品牌和口味偏好、排泄是否正常、驱虫和疫苗是否按时等。仅靠大脑记忆或零散的聊天记录很容易遗忘或混淆，尤其是当家中同时饲养猫和狗时，不同物种的喂养习惯和健康管理周期差异更大。

此外，宠物的洗护管理（洗澡、美容、剪指甲）同样需要记录，方便掌握每只宠物的护理周期。同时，宠物的生日和体重变化也是主人非常关心的信息。

目前市面上缺少一款足够轻量、专注于"多宠日常记录"而非"社交/电商/医疗"的工具。用户需要一个开箱即用、交互简洁的备忘录式应用，能够快速记录每只宠物的日常，并在首页一目了然地看到今日概况。

## Solution

**宠宠记事本** 是一款微信小程序，为多宠家庭提供轻量级的日常记录工具。核心设计原则：

- **单用户免登录** — 直接使用微信身份，无注册流程
- **多宠切换** — 首页可快速切换当前查看的宠物，数据按宠物隔离
- **快速记录** — 喂食、排泄、洗澡、美容、剪指甲均可在两步内完成
- **宠物档案** — 每只宠物拥有独立的档案页，展示照片、性别、品种、生日、体重等信息
- **今日概况** — 首页顶部展示今日喂食次数、排泄次数、待办提醒
- **本地存储** — 所有数据存储在微信本地缓存，即开即用，无需网络

## User Stories（v1.1 新增）

1. 作为宠物主人，我想为每只宠物上传真实照片，以便更直观地识别它们
2. 作为宠物主人，我想记录宠物的性别信息，以便档案更完整
3. 作为宠物主人，我想记录和追踪宠物的体重变化，以便了解健康状况
4. 作为宠物主人，我想在一个页面上集中看到某只宠物的完整档案，包括照片、性别、品种、生日、体重和备注
5. 作为宠物主人，我想在档案页看到该宠物的所有记录统计（喂食、排泄、洗澡等）
6. 作为宠物主人，我想记录洗澡信息（洗护产品、配合程度），以便掌握洗澡周期
7. 作为宠物主人，我想记录美容信息（修毛、修剪等服务项目），以便追踪美容周期
8. 作为宠物主人，我想记录剪指甲信息，以便定期安排
9. 作为宠物主人，我想设置生日提醒，以免忘记毛孩子的生日
10. 作为宠物主人，我想在首页「即将提醒」中看到即将到来的生日提醒

## Implementation Decisions

### 架构

- **全局数据层**：所有数据集中存储在 `app.globalData` 中，通过 `app.js` 提供的 save/get 方法统一读写。
- **持久化**：`wx.setStorageSync` 做本地缓存持久化，在每次数据变更时同步写入。
- **路由方式**：底部 Tab 使用 `wx.redirectTo` 切换，表单页使用 `wx.navigateTo` 入栈。

### 照片处理

- 使用 `wx.chooseMedia` 选择照片，返回本地临时路径
- 照片路径存储在 `pet.photo` 中，直接通过 `wx://` 或本地路径引用
- 注意：临时路径在微信重启后可能失效，建议用户定期更新照片

### 体重追踪

- 每次编辑宠物时可更新体重
- 体重历史存储在 `weightHistory` 数组中，格式 `[{ date: "2024-01-01", weight: "5.0kg" }]`
- 档案页展示当前体重和历史记录列表

## Out of Scope

以下功能明确不在 v1.1 范围内：

- **云开发迁移** — 将数据从本地缓存迁移到微信云开发
- **推送通知** — 通过微信订阅消息发送提醒
- **健康报告** — 生成周报/月报
- **体重曲线图** — 可视化体重变化图表
- **数据备份与导出** — 将记录导出为 JSON/CSV
- **主题切换** — 自定义界面主题色
- **多用户** — 暂不支持多用户登录

## Further Notes

- 该小程序为纯个人使用场景设计，不需要用户系统、权限管理和多用户隔离
- v1.1 新增的洗护记录和生日提醒，补全了养宠日常的完整管理闭环
- 宠物照片使用微信临时路径，是权衡了存储简便性与数据持久性的设计方案
- 项目的 `agent.md` 文件和此 PRD 保持了同步
